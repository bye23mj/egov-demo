# 공공표준용어 생성을 위한 공통표준도메인 기준 추출본

- 원본: 공공데이터베이스 표준화 관리 매뉴얼 2026.4
- 추출 범위: 원문 인쇄 기준 130p~147p
- 목적: Codex 또는 Claude Code에서 공공표준용어 생성 시 `공통표준도메인`을 판단하고, 데이터 타입/길이/저장형식/표현형식/허용값을 결정하는 기준으로 사용
- 핵심 범위: 공통표준도메인 개요, 관리항목, 정의 절차, 도메인그룹명, 도메인분류명, 공통표준도메인명, 데이터타입/길이/소수점길이, 저장형식, 표현형식, 단위, 허용값

---

## 1. 사용 지침

Codex가 공공표준용어 후보를 생성할 때는 아래 순서로 판단한다.

1. 입력된 한글 컬럼명, 속성명, 용어명을 공통표준단어 단위로 분할한다.
2. 분할된 단어 중 마지막 단어가 데이터의 형식, 길이, 표현방식, 값 범위를 식별하는 `형식단어`인지 확인한다.
3. 형식단어를 기준으로 `공통표준도메인분류명`을 매핑한다.
4. 공통표준도메인명은 원칙적으로 `공통표준도메인분류명 + 데이터타입 약어 + 길이` 형태로 판단한다.
   - 예: `연도C4`, `연월C6`, `연월일C8`, `주민등록번호C13`, `전화번호V11`, `금액N15`
5. 데이터타입은 다음 약어 기준을 적용한다.
   - `C`: CHAR
   - `V`: VARCHAR
   - `N`: NUMERIC
   - `D`: DATETIME
6. 길이에 소수점이 있는 경우 `데이터길이,소수점길이` 형태로 표현한다.
   - 예: `경도N13,10`, `위도N12,10`, `면적N19,9`, `금액N22,2`
7. 날짜/시간 도메인은 기관 DB 특성에 따라 `연월일시분초D` 또는 `연월일시분초C14` 중 선택할 수 있다.
8. 데이터베이스 저장형식은 표현용 특수문자를 제외하고 원본 데이터만 저장하는 형식을 우선한다.
   - 예: `주민등록번호C13` 저장형식은 `13자리 문자`, 표현형식은 `999999-9999999`
9. 화면/API/문서 표현형식은 `-`, `:`, 공백, 콤마 등 일반적인 구분문자를 사용할 수 있으나, 저장형식과 혼동하지 않는다.
10. 암호화 대상 도메인은 원본 데이터의 도메인을 우선 정의한다. 암호화된 결과값 저장 컬럼은 범용 도메인 `암호화번호V256`을 적용할 수 있다.
11. 도메인 적용 시 한글 데이터 길이는 원칙적으로 2byte 기준으로 산정하되, UTF-8 등 3byte 환경에서는 오버플로우 여부를 검토하여 조정한다.
12. 허용값은 공통표준도메인에서는 포괄적으로 정의하고, 실제 공통표준용어 정의 시 더 상세화할 수 있다.

## 2. 출력 권장 형식

공공표준용어 후보 생성 시 다음 표 형식을 우선 사용한다.

```markdown
| 입력 컬럼명 | 분할 단어 | 형식단어 | 도메인그룹명 | 도메인분류명 | 공통표준도메인명 | 데이터타입 | 데이터길이 | 소수점길이 | 저장형식 | 표현형식 | 허용값 | 검토의견 |
|---|---|---|---|---|---|---|---:|---:|---|---|---|---|
| 가입일자 | 가입 / 일자 | 일자 | 날짜/시간 | 연월일 | 연월일C8 | CHAR | 8 | - | YYYYMMDD | YYYY-MM-DD | 유효한 날짜 | 마지막 단어가 형식단어이므로 도메인 적용 가능 |
| 신청자주민등록번호 | 신청자 / 주민등록번호 | 주민등록번호 | 번호 | 주민등록번호 | 주민등록번호C13 | CHAR | 13 | - | 13자리 문자 | 999999-9999999 | - | 식별번호 성격이므로 번호 도메인 적용 |
| 사용여부 | 사용 / 여부 | 여부 | 코드 | 여부 | 여부C1 | CHAR | 1 | - | 문자1자리 | Y/N | Y,N | 긍정/부정 코드 도메인 적용 |
```

## 3. 공통표준도메인 개념

- 도메인은 데이터 값의 일반적인 특징을 분류한 것으로, 속성이 취할 수 있는 값의 집합이다.
- 도메인 표준화는 기관에서 생성되는 데이터의 속성을 분류하여 도메인 분류체계를 수립하고, 도메인별 데이터 타입과 길이 등 구성요소를 정의하는 작업이다.
- 공통표준도메인은 여러 공공기관에서 사용하는 공통표준용어의 데이터 속성을 분석하여 정의한 범정부 차원의 상위수준 도메인이다.
- 공통표준도메인은 공통표준용어 구성 시 용어 끝의 형식단어를 사용하여 정의된다.
- 공통표준도메인을 적용하면 공통표준용어의 의미를 명확히 하고 데이터 형식의 일관성을 확보할 수 있다.

### 적용 예시

| 공통표준용어 | 공통표준단어 구성 | 공통표준도메인명 | 효과 |
|---|---|---|---|
| 신청자주민등록번호 | 신청자 + 주민등록번호 | 주민등록번호C13 | 공통표준용어의 데이터 형식 일관성 확보 |
| 등록일자 | 등록 + 일자 | 연월일C8 | 공통표준용어의 데이터 형식 일관성 확보 |

## 4. 공통표준도메인 관리항목

공통표준도메인은 다음 관리항목으로 구성한다.

| 관리항목 | Codex 처리 기준 |
|---|---|
| 공통표준도메인그룹명 | 데이터가 가질 수 있는 형식을 기준으로 최상위 그룹을 분류한다. 업무별로 다르게 정의하지 않는다. |
| 공통표준도메인분류명 | 데이터 타입과 길이 등을 정의하는 구체적인 명칭을 사용한다. 예: 번호가 아니라 주민등록번호, 법인등록번호 |
| 공통표준도메인명 | `도메인분류명 + 데이터타입약어 + 길이` 형태로 생성한다. 예: 연월일C8, 금액N15 |
| 공통표준도메인 설명 | 도메인의 의미, 규칙, 구성요소를 명확히 설명한다. |
| 데이터타입 | CHAR, VARCHAR, NUMERIC, DATETIME 등을 적용한다. |
| 데이터길이 | 데이터 값이 가질 수 있는 자릿수 또는 문자 길이를 정의한다. |
| 데이터소수점길이 | NUMERIC 타입 중 소수부가 필요한 경우 정의한다. |
| 저장형식 | DB에 저장되는 원본 데이터 형식을 정의한다. 표현용 특수문자는 원칙적으로 제외한다. |
| 표현형식 | 화면, API, 문서 등 외부 표현 시 사용할 표시 형식을 정의한다. |
| 단위 | 법정단위 또는 국가표준 단위를 준용한다. |
| 허용값 | 최대/최소값 또는 유효값을 정의한다. 공통표준용어 정의 시 상세화 가능하다. |

## 5. 공통표준도메인 정의 절차

### 5.1 공통표준도메인그룹명 정의

- 공통표준단어 표준화 작업에서 도출된 형식단어를 분석한다.
- 공통적인 데이터 형식과 속성에 따라 공통표준도메인그룹명을 정의한다.
- 그룹명은 최상위 분류이므로 너무 상세하게 정의하지 않는다.

### 5.2 공통표준도메인분류명 및 공통표준도메인명 정의

- 특정 데이터 형식과 속성을 대상으로 공통표준도메인분류명을 정의한다.
- 해당 분류에 데이터 타입과 길이를 결합하여 공통표준도메인명을 정의한다.

| 공통표준용어명 | 형식단어 | 도메인그룹명 | 도메인분류명 | 공통표준도메인명 |
|---|---|---|---|---|
| 법인등록번호 | 법인등록번호 | 번호 | 법인등록번호 | 법인등록번호C13 |
| 신청자주민등록번호 | 주민등록번호 | 번호 | 주민등록번호 | 주민등록번호C13 |
| 자택전화번호 | 전화번호 | 번호 | 전화번호 | 전화번호V11 |
| 가입연도 | 연도 | 날짜/시간 | 연도 | 연도C4 |
| 가입일시 | 일시 | 날짜/시간 | 연월일시분초 | 연월일시분초D |
| 가입일자 | 일자 | 날짜/시간 | 연월일 | 연월일C8 |

### 5.3 공통표준도메인 관리항목 정의

- 도출된 공통표준도메인명에 대해 데이터 타입, 데이터 길이, 소수점 길이, 저장형식, 표현형식, 단위, 허용값을 정의한다.

## 9. 저장형식, 표현형식, 허용값 판단 기준

### 9.1 데이터타입과 길이

| 구분 | 데이터타입 | 길이 | 소수점길이 | 설명 |
|---|---|---|---|---|
| 문자형 | CHAR | 숫자 | 해당 없음 | 고정 길이 데이터타입. 짧은 값 입력 시 나머지 공간은 공백으로 채움 |
| 문자형 | VARCHAR | 숫자 | 해당 없음 | 가변 길이 데이터타입. 짧은 값 입력 시 나머지 공간을 채우지 않음 |
| 숫자형 | NUMERIC | 숫자 | 숫자 | 소수점을 포함한 전체 길이를 데이터길이로 표현 |
| 날짜형 | DATETIME | 해당 없음 | 해당 없음 | 연월일시분초를 모두 표현할 수 있는 데이터타입 |

### 9.2 저장형식

- DB 저장 시 사용하는 원본 데이터 형식이다.
- 원칙적으로 표현용 특수문자, 하이픈, 콜론, 콤마 등은 배제한다.
- 데이터길이와 소수점길이는 저장형식에 따라 결정한다.

### 9.3 표현형식

- 화면, 애플리케이션, 문서, API 등에 데이터를 표시할 때의 형식이다.
- 일반적인 구분문자 `,`, `:`, 공백, `-` 등을 사용할 수 있다.
- 표현형식은 저장형식과 다를 수 있다.

### 9.4 단위

- 국제도량형기구 및 국제법정계량기구의 국제협약에 따라 국가에서 정한 법정단위를 준용한다.

### 9.5 허용값

- 도메인이 가질 수 있는 최대값, 최소값, 유효값을 정의한다.
- 공통표준도메인에서는 포괄적으로 정의하고, 개별 공통표준용어에서 상세화할 수 있다.
