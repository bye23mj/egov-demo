# data-structure-verification (공공DB 데이터구조 검증)

> da-agent 3단계를 독립 스킬로 승격. 표준화검토(metadata-agent)를 반영한 물리모델의 구조 건전성을 검증한다.
> ECC `database-migrations`(안전변경)·`database-reviewer`(스키마검토) 패턴을 참고하되 **공공DB·대상 DBMS 기준**으로 점검한다.

## 사용 시점

- da-agent 2단계(표준화검토) 직후, 4단계(산출물관리) 전의 검증 게이트
- "데이터구조/스키마 검증", "정규화·무결성 점검"을 요청받을 때

## 입력

- 표준 반영 물리모델(엔터티/테이블·컬럼·타입·길이·키·관계)
- `대상 DBMS`: Oracle | PostgreSQL (기본 Oracle 11g)
- metadata-agent 표준검토 결과(`{요구사항번호}-표준검토결과`)

## 검증 항목

| 범주 | 점검 내용 |
|---|---|
| 정규화 | 이상현상(삽입/갱신/삭제), 반복·다중값 속성, 부분/이행 함수종속 |
| 식별자 | 주식별자 유일성·최소성, 후보/외래식별자 구분, 인조키 적정성 |
| 참조무결성 | FK ↔ PK 도메인 일치, 관계 차수·선택성, 순환참조 |
| 도메인 | 타입·길이·NULL 허용·기본값이 표준도메인과 일치 |
| 코드 | `_CD` 컬럼의 표준코드 매핑·허용값 정의 |
| 명명 | 표준용어 영문약어명 기준, 일관성 |
| 테이블명 | [`.claude/rules/테이블명명규칙.md`](../../rules/테이블명명규칙.md) 준수(지연 로딩) — 구조(`[업무영역]+[관리대상]+[유형]`)·유형 접미어·엔터티 정합·금지명(DATA/INFO/MASTER/TEMP/ETC/TEST/OLD/NEW) 회피. 동 문서 10절 12개 체크리스트로 검토 |
| 제약/인덱스 | PK/UK/NOT NULL/인덱스 적정성, 과다/누락 |

## DBMS별 추가 점검

| 항목 | Oracle 11g | PostgreSQL |
|---|---|---|
| 식별자 길이 | 30바이트 이내 초과 여부 | 63바이트 이내 초과 여부 |
| 타입 호환 | VARCHAR2/NUMBER/DATE 매핑 정합 | VARCHAR/NUMERIC/TIMESTAMP 매핑 정합 |
| 채번 | SEQUENCE 설계 여부 | IDENTITY/SERIAL 설계 여부 |
| 페이징 전제 | ROWNUM 가능 구조 | LIMIT/OFFSET 가능 구조 |

## 판정 및 환류

- 각 항목을 `적합 / 보완 / 부적합`으로 판정한다.
- `보완`/`부적합`은 사유와 함께 **1단계(모델링) 또는 2단계(표준검토)로 환류**한다.
- 전 항목 `적합` 전까지 4단계(산출물관리) 진행을 보류한다.

## 산출물

- 구조검증 결과표(범주·항목·판정·사유·환류대상) → 산출물의 `표준검토상태` 및 품질관리(NCR)와 연계

## 경계

- 표준 자체를 판정·신설하지 않는다(metadata-agent 결과 인용).
- 모델을 직접 수정하지 않는다(검증·환류 권고까지).
