# Slack Agent Orchestrator (Slack 명령 → 전문 에이전트 라우팅)

Slack의 짧은 명령어와 첨부파일을 받아 **적합한 전문 에이전트로 라우팅 → 검토 → Markdown 결과 → Slack 보고**까지 조율하는 통합 스킬. 판단 주체·원칙은 `slack-agent`가, 절차·라우팅·게이트는 이 스킬이 정의한다.

## 실행 모드: 분석 → 단일 위임 (라우팅)

- 명령·첨부를 분석해 **하나의 전문 에이전트**(또는 소수)를 선택하고 그 역할로 검토한다.
- 전문 에이전트는 `Agent(subagent_type)` 또는 검토 스킬 `Skill`로 호출한다.
- 진행 상태는 Slack에 단계적으로 공유한다(da_progress_notify 패턴 재사용 가능).

## 작업공간 (절대 경로)

```text
docs/06. slack-review/{YYYYMMDD_HHMMSS}_{명령}/
├── attachments/                 # 다운로드한 첨부 원본
├── extracted/                   # 첨부에서 추출한 텍스트(분석용)
├── {명령}-검토결과.md            # 7) Markdown 검토 결과
└── _slack-summary.md            # 7) Slack 요약 메시지 원문(발송 전 검토)
```

## 라우팅 표 (명령어 + 첨부 신호 → 전문 에이전트)

| 명령 키워드 | 첨부 신호(파일명/내용) | 전문 에이전트 | 검토/수행 기준(위임) |
|---|---|---|---|
| **데이터모델링** (생성·구축) | 요구사항정의서·`{요구사항번호}-db-modeling.md`·테이블정의서 | **da-agent → dba-agent** (체이닝) | da-agent로 모델·산출물 7종 생성 → dba-agent로 **실제 테이블 생성 완료** |
| 데이터모델링검토 · ERD검토 · 모델검토 | 데이터모델링정의서·ERD·엔터티/속성정의서 | **da-agent** (`data-modeling`+`data-structure-verification`) | 정규화·식별자·관계·도메인·표준 정합 |
| 표준화검토 · 용어검토 · 표준검토 | 표준용어/단어/도메인/코드 | **metadata-agent** (`metadata-standardize`) | 표준용어·단어·도메인·코드 매핑 |
| DB검토 · 테이블검토 · DDL검토 | 테이블/컬럼정의서·DDL·`*.sql` | **dba-agent** (`db-object-creation`) | DDL·제약·인덱스·테이블명명규칙 |
| 코드검토 · 파이썬검토 | `*.py` | **python-reviewer** | PEP8·타입힌트·보안·성능 |
| 설계검토 · 아키텍처검토 | 설계서·아키텍처·시퀀스 | **design-reviewer** | 계층분리·결합도·패턴·확장성 |
| 보안검토 | 코드·설정·`*.properties` | **security-auditor** | 취약점·시크릿·인증·인가 |
| 품질검토 · 정적분석 | 코드 전반 | **static-analyzer** | 정적 결함·복잡도·중복 |

- **체이닝 명령(데이터모델링)**: 단일 검토가 아니라 **생성→구축→등록 파이프라인**이다.
  1. **da-agent**(da-orchestrator 1~7): 첨부 요구사항으로 모델링·표준화·검증·산출물 7종 생성.
  2. **dba-agent**(dba-orchestrator): 생성된 테이블정의서·컬럼정의서로 DDL 생성 → **대상 DB(기본 Oracle 11g/komsa)에 테이블 실제 생성**(기존 시 재생성). DROP/CREATE 등 실 DB 변경이므로 대상 계정·재생성 여부를 보고에 명시한다.
  3. **산출물 Confluence 등록(상시)**: da-agent 산출물을 `scripts/upload_pages_in_folders.py`로 등록(da-orchestrator Phase 9)하고, **등록된 Confluence 위치(URL)를 Slack에 공유**한다. 예: `https://nsonesoft.atlassian.net/wiki/spaces/TNYUU/folder/677871627`.
  4. 각 단계 진행을 Slack에 공유하고, 최종에 **생성 산출물 + 생성된 테이블 목록 + Confluence 등록 위치(URL)** 를 보고한다.
- **복수 후보** 시: 명령 키워드를 1순위, 첨부 다수 유형을 보조 근거로 **주 에이전트 1개**를 정하고, 부수 검토가 필요하면 보조 에이전트를 후속 제안한다.
- **표에 없음/모호**: 추측하지 말고 6)에서 Slack에 1회 확인 질문을 올린다.

## 첨부 내용 추출 (유형별)

| 유형 | 추출 방법 |
|---|---|
| `.xlsx`/`.xls` | `python3`+openpyxl 로 시트·헤더·표본행 추출 → `extracted/` 텍스트 |
| `.docx` | `unzip -p {file} word/document.xml` 후 태그 제거(또는 python-docx) |
| `.pdf` | `Read` 도구(PDF 페이지 읽기) |
| `.csv`/`.tsv`/`.md`/`.txt`/`.sql` | `Read`/`Grep` 직접 |
| 기타/바이너리 | 파일명·메타만 사용, 본문 미해석 명시 |

> 시크릿(토큰·비밀번호·주민번호 등)은 추출물·요약·로그에 **마스킹**한다.

## 워크플로우 (8단계)

### 1) Slack 명령어 확인
- 멘션/명령(`@GovAI {명령}`)을 파싱한다. Slack MCP `slack_read_channel`/`slack_read_thread`로 메시지·스레드를 읽는다.
- 명령 키워드를 라우팅 표 후보로 정규화한다.

### 2) 첨부파일명 확인
- 메시지의 첨부 목록(파일명·확장자·크기)을 수집한다. 파일명 패턴을 라우팅 보조 신호로 본다.

### 3) 첨부파일 내용 확인
- `slack_read_file`/다운로드로 `attachments/`에 저장 → 유형별 추출 표대로 `extracted/`에 텍스트화한다.
- 내용에서 문서 종류 신호(엔터티/표준용어/DDL/코드 등)를 식별한다.

### 4) 실행할 Agent 결정
- 1)+2)+3) 종합으로 **주 에이전트 1개**를 선택한다. 선택 근거(키워드·파일·내용)를 기록한다.
- 모호하면 6)로 가서 Slack 확인 질문(이후 중단·대기).

### 5) Agent별 검토 기준 분석
- 선택 에이전트를 호출한다: `Agent(subagent_type: "{에이전트}")` 또는 검토 스킬 `Skill`.
- 입력: `extracted/` 텍스트 + 첨부 메타. 출력: 항목별 `적합/보완/부적합` 판정·사유·근거.
- 결과를 `{명령}-검토결과.md`로 작성한다(아래 형식).

### 6) 실행 에이전트 상태 공유
- 시작·선택·진행·완료를 Slack에 단계적으로 공유한다.
  `python3 scripts/da_progress_notify.py step --reqid {명령} --step "{에이전트} 검토" --status in_progress|completed --note "{요지}"`
- 모호·권한·추출 실패 시 상태와 함께 확인 질문/한계를 Slack에 올린다.

### 7) Slack 요약 메시지 작성
- `_slack-summary.md`에 요약을 작성하고 Slack에 게시한다(`slack_send_message`). 요약은 **핵심 판정·주요 지적 3~5건·첨부별 상태**로 압축한다. 원문 대량 인용 금지.

### 8) 다음 액션 제시
- 보완 필요 항목, 책임자, 후속 에이전트(예: da→dba 구축, 산출물 Confluence 등록 Phase 9) 등 **명확한 다음 단계**를 요약 말미에 제시한다.

> **전략적 compact 체크포인트** ([[strategic-compact]]): 검토결과는 `_검토결과.md`로 보존된다. 한 Slack 명령 처리를 끝내고 **서로 무관한 다음 명령/첨부**로 넘어가기 전(예: 코드검토 → 표준화검토)에는 직전 첨부 본문·추출 내용이 컨텍스트를 오염시키므로 `/compact`를 제안한다. 동일 명령의 후속 보완 작업 중에는 하지 않는다.

## `{명령}-검토결과.md` 형식

```markdown
# {명령} 검토 결과 — {YYYY-MM-DD}

## 라우팅
- 선택 에이전트: {에이전트} (근거: 명령 "{키워드}" + 첨부 {파일})
- 검토 첨부: {파일 목록}

## 종합 판정
- {적합 / 보완 N건 / 부적합 M건}

## 항목별 검토
| 첨부 | 범주 | 판정 | 지적/사유 | 권고 |
|---|---|---|---|---|

## 주요 지적 (Top 3~5)
1. …

## 다음 액션
- [ ] …
```

## Slack 요약 형식 (예)

```text
:mag: {명령} 검토 완료 — {에이전트}
• 종합: 보완 3건 (부적합 0)
• 핵심: ① 식별자 중복 ② 표준 미적용 컬럼 4 ③ 관계 누락
• 첨부: 데이터모델링정의서 ✅ / 테이블정의서 ⚠️보완 / ERD ✅
:arrow_right: 다음: 보완 후 dba-agent로 DB 구축 / 산출물 Confluence 등록
(상세: docs/06. slack-review/.../{명령}-검토결과.md)
```

## 에러 핸들링

| 상황 | 전략 |
|---|---|
| 명령/의도 모호 | Slack 확인 질문 1회 후 대기(추측 검토 금지) |
| 라우팅 후보 다수 | 주 에이전트 1개 선택 + 보조 검토 후속 제안 |
| 첨부 추출 실패(암호·손상) | 해당 첨부 `검토불가` 명시, 나머지 진행 |
| 첨부에 시크릿 포함 | 마스킹 후 진행, 보안 노출 경고를 요약에 포함 |
| 권한 없는 채널/파일 | 접근 중단·사유 보고, 권한 요청 안내 |
| 전문 에이전트 실패 | 1회 재시도, 재실패 시 부분 결과+한계 보고 |

## 테스트 시나리오

1. 입력: `@GovAI 데이터모델링검토` + 데이터모델링정의서.xlsx·테이블정의서.docx·ERD.pdf.
2. 1~3) 명령·첨부명·내용 확인 → 4) **da-agent** 선택(근거 기록).
3. 5) da-agent로 정규화·식별자·관계·표준 검토 → `데이터모델링검토-검토결과.md`.
4. 6) 진행 상태 Slack 공유 → 7) 요약 게시 → 8) "보완 후 dba 구축/Confluence 등록" 다음 액션.

## 경계

- 검토 기준을 직접 확정하지 않는다(전문 에이전트 위임).
- 첨부 원본 수정·삭제 금지(검토·보고까지).
- 시크릿·개인정보를 Slack/로그/커밋에 노출하지 않는다(마스킹).
- 권한 없는 채널·파일 접근/게시 금지.
