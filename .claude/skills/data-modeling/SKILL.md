# data-modeling (공공DB 데이터모델링)

> da-agent 1단계를 독립 스킬로 분리. 요구사항을 표준화·검증·산출물 단계의 기준이 되는 데이터 모델로 변환한다.
> 표준용어·도메인 확정은 하지 않는다(2단계 metadata-agent 위임). 여기서는 **후보**를 도출한다.

## 사용 시점

- da-agent 1단계(데이터모델링) 진입 시
- "데이터 모델링", "엔터티/속성 도출", "db-modeling 작성"을 요청받을 때
- 표준화검토(2단계) 전, 모델 초안이 필요할 때

## 입력

- 요구사항정의서 · 화면설계서 · 인터페이스정의서 · 업무규칙정의서
- (있으면) 기존 DDL · 테이블/컬럼정의서
- **대상 DBMS**: Oracle | PostgreSQL (기본 Oracle 11g) — 이후 단계 분기의 기준

## 절차

### 1. 요구사항 분석
- 명사(엔터티 후보)·동사(관계 후보)·규칙(제약 후보)을 식별한다.
- 화면/인터페이스 항목에서 속성 후보와 코드성 항목을 추출한다.

### 2. 논리모델 도출
- 엔터티·속성·관계·식별자(주/후보/외래)를 정의한다.
- 중복·반복·다중값 속성을 식별하고 정규화 후보를 표시한다.
- 관계 차수·선택성, 수퍼/서브타입을 명시한다.

### 3. 물리모델 후보
- 엔터티→테이블, 속성→컬럼으로 변환한다(한글명 유지, 영문명은 **후보**).
- **테이블명(한글/영문)은 [`.claude/docs/standards/테이블명명규칙.md`](../../docs/standards/테이블명명규칙.md)를 참고하여 작성한다**(이 단계 진입 시에만 Read — 지연 로딩). 핵심: `[업무영역]+[관리대상/엔터티]+[테이블유형]` 구조, 영문은 표준단어 영문약어명 기반 `TB_[업무영역]_[관리대상]_[유형]`, 유형 접미어(`_HIST`/`_LOG`/`_CD`/`_MAP`/`_STAT`/`_TMP` 등)로 유형 식별, `DATA`/`INFO`/`MASTER`/`TEMP`/`ETC`/`TEST`/`OLD`/`NEW` 등 금지명 회피, 엔터티명과 정합.
- 데이터 타입/길이는 후보로만 기재한다(확정은 2단계 표준도메인).
- PK/FK/UK/Index/NOT NULL, 테이블·컬럼 COMMENT 후보를 정리한다.
- 코드성 속성(`_CD` 후보)·개인정보/민감정보 후보를 표시한다.

### 4. 산출물 작성
`{요구사항번호}-db-modeling.md`를 작성한다(2~7단계 기준 입력).

## 산출물 형식 (`{요구사항번호}-db-modeling.md`)

```markdown
# 데이터 모델링 — {요구사항번호}

## 개요
- 대상 DBMS: Oracle 11g | PostgreSQL
- 업무영역 / 주제영역 / DB명

## 엔터티 목록
| 엔터티명 | 설명 | 주식별자 | 수퍼타입 | 개인정보 |

## 속성 목록
| 엔터티 | 한글속성명 | 영문명(후보) | 타입(후보) | 길이(후보) | 식별자 | 필수 | 코드성(_CD) | 개인정보 |

## 관계 목록
| 부모 | 자식 | 차수 | 식별/비식별 | 설명 |

## 물리 테이블/컬럼 후보
| 한글테이블 | 영문테이블(후보) | 한글컬럼 | 영문컬럼(후보) | PK/FK/UK | NOT NULL |

## 업무규칙
- (제약·검증·도출 규칙)

## 표준화 검토 요청 항목
- (2단계 metadata-agent로 넘길 컬럼/코드 목록)
```

## 경계

- 표준용어·도메인·코드를 **확정하지 않는다**(후보만, 확정은 metadata-agent).
- 요구사항에 없는 엔터티/속성을 임의 생성하지 않는다. 모호하면 `보완 필요`로 표기한다.
- 산출물 xlsx는 생성하지 않는다(4단계 data-output-management 담당).
