# data-analysis (공공DB 데이터 분석)

> da-agent 7단계를 독립 스킬로 승격. 검증·산출물 단계를 통과한 데이터구조를 분석하여 활용·운영 인사이트를 도출한다.

## 사용 시점

- da-agent 4단계(산출물관리) 이후 마무리 분석 단계
- "데이터 분석", "프로파일링", "표준화 효과 분석"을 요청받을 때

## 입력

- 확정 물리모델·정의서(테이블/컬럼정의서)
- metadata-agent 표준검토 결과, 데이터구조검증 결과
- (있으면) 기존 데이터/통계, 요구사항의 규모 가정

## 분석 항목

| 항목 | 내용 |
|---|---|
| 데이터 프로파일링 | 컬럼별 도메인·NULL 허용·코드값 후보 분포, 카디널리티 추정 |
| 규모/증가율 | 엔터티별 예상 행수·연증가율, 대용량 테이블 식별 |
| 코드값 분포 | `_CD` 컬럼의 코드값 다양성·기타성(zz/99) 필요성 |
| 개인정보 영향 | 개인정보/민감정보 컬럼 범위, 암호화 도메인 적용 영향 |
| 연계·통합 | 주제영역 간 공유 엔터티·표준코드 연계 포인트, 기관 간 연계 가능성 |
| 표준화 효과 | 표준 적용으로 인한 중복 컬럼 감소·명명 일관성·연계 용이성 정량/정성 |

## 절차

1. 정의서·표준검토 결과를 로드해 컬럼·도메인·코드 메타를 집계한다.
2. 규모 가정과 결합해 대용량·고증가 테이블을 식별하고 인덱싱/파티셔닝 후보를 제안한다.
3. 개인정보·연계·표준화 효과를 정리한다.
4. 분석 요약을 산출물관리대장 또는 별도 분석 노트에 기록한다.

## 산출물

- `{요구사항번호}-데이터분석노트.md`(또는 산출물관리대장 분석 섹션): 프로파일링·규모·개인정보·연계·표준화 효과 요약 + 운영 권고(인덱스/파티셔닝/보존기간)

## 경계

- 실제 운영 데이터에 쿼리하지 않는다(모델·정의 기반 분석·추정).
- 분석 권고일 뿐 스키마/데이터를 변경하지 않는다.
