# data-quality-management (공공DB 데이터 품질관리)

> ECC `quality-nonconformance` 스킬의 NCR·CAPA·RCA·SPC 방법론을 **공공데이터베이스 데이터 품질**에 적용하도록 현지화한다.
> 방법론 원본(규제 제조 품질 전반): [`../quality-nonconformance/SKILL.md`](../quality-nonconformance/SKILL.md) 를 필요 시 참조한다.

## 사용 시점

- da-agent 6단계(품질관리)에서 산출물·데이터구조의 품질을 점검할 때
- "데이터 품질 점검", "부적합 등록", "시정조치"를 요청받을 때
- 표준검토·구조검증 후 최종 품질 게이트를 운영할 때

## 데이터 품질 차원 (점검 항목)

| 차원 | 점검 내용 |
|---|---|
| 표준 준수성 | 표준용어/영문약어명/도메인/코드 적용 여부(metadata-agent 결과 대비) |
| 완전성 | 필수항목(주식별자·표준검토상태·개인정보여부 등) 결측 |
| 유효성 | 도메인(타입·길이·허용값) 위반, 코드값(`_CD`) 미정의/오류 |
| 일관성 | ERD ↔ 정의서 ↔ 메타데이터 ↔ 마이그레이션 간 불일치 |
| 정확성 | 업무규칙·참조무결성 위반 |
| 보호성 | 개인정보/민감정보 표기·암호화 도메인 누락 |

## NCR 라이프사이클 (ECC 현지화)

```text
부적합 발견 → NCR 등록 → 격리/표시(보완·부적합) → 근본원인분석(RCA)
            → CAPA 정의 → 시정 반영(3·4단계 환류) → 효과 검증 → 종결
```

- **NCR 등록**: 부적합 항목을 `데이터품질부적합대장`에 1행으로 기록(대상 산출물·컬럼·차원·심각도·발견일).
- **RCA**: 5 Why / 특성요인도(Fishbone)로 근본원인 도출(예: 표준 미적용, 모델링 누락, DBMS 분기 오류).
- **CAPA**: 시정조치(해당 산출물 수정) + 예방조치(체크리스트·규칙 보강).
- **검증**: 시정 후 재점검하여 `적합` 전환 확인. 미결 시 da-agent 완료 보고 금지.

## 심각도 분류

| 등급 | 기준 | 처리 |
|---|---|---|
| 치명(Critical) | 개인정보 보호 누락, 참조무결성 붕괴 | 즉시 차단·시정 |
| 중대(Major) | 표준 미준수, 필수항목 결측 | 시정 후 진행 |
| 경미(Minor) | 표기·설명 보완 | 보완 권고 |

## 산출물

- `{요구사항번호}-데이터품질부적합대장.xlsx` (NCR 목록: 부적합ID·대상·차원·심각도·원인·CAPA·상태)
- 품질 점검 결과를 산출물관리대장의 `표준검토상태`/비고와 연계

## 경계

- 표준 적합성 판정 기준은 metadata-agent 결과를 따른다(자체 표준 신설 금지).
- 데이터 자체를 수정하지 않는다(산출물·정의 부적합 식별·시정 권고까지).
