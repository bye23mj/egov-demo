# metadata-standardize

> 메타데이터 표준화 워크플로우. 컬럼을 공공표준으로 검토하고, 없으면 기관표준을 생성하여 DB 설계에 반영한다.
> 판단 주체·행동 원칙은 `metadata-agent` 에이전트가, 절차·도구·출력은 이 스킬이 정의한다.

## 사용 시점

- DA agent의 `{요구사항번호}-db-modeling.md`(DB 모델링 초안)를 받아 컬럼을 표준화할 때
- "메타데이터 등록/검토/수정", "표준용어 검토", "컬럼 표준화"를 요청받았을 때
- 단일 컬럼을 표준화하여 기관메타데이터에 등록할 때

---

## 메타데이터 자산 (TSV 인덱스)

조회·등록은 모두 TSV로 한다. xlsx는 보관용이다.

- **공공: xlsx = 원본, tsv = 파생 인덱스** (`scripts/build_metadata_index.py`, xlsx→tsv). READONLY — 수정 금지.
- **기관: tsv = 원본(source of truth), xlsx = 파생** (`scripts/sync_metadata.py`, tsv→xlsx).

### 공공메타데이터 (READONLY · 조회 전용)

| TSV 인덱스 (조회) | 원본 (보관) | 행수 |
|---|---|---|
| `docs/03. metadata/공공/공공표준단어.tsv` | `공공표준단어.xlsx` | ~3,285 |
| `docs/03. metadata/공공/공공표준용어.tsv` | `공공표준용어.xlsx` | ~13,177 |
| `docs/03. metadata/공공/공공표준도메인.tsv` | `공공표준도메인.xlsx` | ~130 |
| `docs/03. metadata/공공/공공표준코드.tsv` *(없으면 미생성)* | `공공표준코드.xlsx` | - |

### 기관메타데이터 (생성·수정 가능)

현재 기관 도메인은 **KOMSA**(`{도메인}` = `KOMSA`). 등록은 해당 TSV에 **행 추가(append)**.

| TSV (조회·등록) | 비고 |
|---|---|
| `docs/03. metadata/기관/KOMSA표준단어.tsv` | 헤더만 있는 빈 템플릿 → 신규 단어 append |
| `docs/03. metadata/기관/KOMSA표준용어.tsv` | 신규 용어 append |
| `docs/03. metadata/기관/KOMSA표준도메인.tsv` | 신규 도메인 append |
| `docs/03. metadata/기관/KOMSA표준코드.tsv` *(없음)* | 표준코드 신규 발생 시 코드 스키마 헤더로 새로 생성 |

### TSV 컬럼 스키마 (탭 구분, 1행=헤더)

```text
단어   : 번호 · 공통표준단어명 · 영문약어명 · 영문명 · 설명 · 형식단어여부 · 도메인분류명 · 이음동의어목록 · 금칙어목록 · 제정차수 · 개정구분명 · 개정항목 · 개정사유
도메인 : 번호 · 그룹명 · 분류명 · 도메인명 · 설명 · 데이터타입 · 데이터길이 · 소수점길이 · 저장형식 · 표현형식 · 단위 · 허용값 · 제정차수 · 개정구분명 · 개정항목 · 개정사유
용어   : 번호 · 공통표준용어명 · 설명 · 영문약어명 · 도메인명 · 허용값 · 저장형식 · 표현형식 · 행정표준코드명 · 소관기관명 · 이음동의어목록 · 제정차수 · 개정구분명 · 개정항목 · 개정사유
코드   : 관리부서명 · 한글코드명 · 영문코드명 · 코드설명 · 데이터타입 · 데이터길이 · 코드값 · 코드값의미
```

### 조회 방법 (Grep)

```bash
# 표준용어 정확매칭 (2번째 컬럼 = 용어명)
grep -P "^[^\t]*\t사용자등록일자\t" "docs/03. metadata/공공/공공표준용어.tsv"
# 표준단어 정확매칭 (2번째 컬럼 = 단어명)
grep -P "^[^\t]*\t사용자\t" "docs/03. metadata/공공/공공표준단어.tsv"
# 표준도메인 (4번째 컬럼 = 도메인명)
grep -P "\t연월일C8\t" "docs/03. metadata/공공/공공표준도메인.tsv"
```

## 규칙 문서는 지연 로딩 (토큰 절약)

표준화 판단 규칙은 아래 4개 문서를 근거로 삼는다. **4개를 미리 한꺼번에 읽지 않는다.** 각 표준 대상 단계에 실제로 진입할 때, 그 단계에 해당하는 문서 1개만 `Read`한다. 같은 세션에서 이미 읽은 문서는 다시 읽지 않는다. 공공표준 조회만으로 매칭되는 컬럼은 규칙 문서를 읽을 필요가 없다.

| 표준 대상 단계 | 그 단계에서만 Read할 규칙 문서 |
|---|---|
| 표준단어 생성 | `.claude/rules/공공표준단어생성.md` |
| 표준용어 생성 | `.claude/rules/공공표준용어생성.md` |
| 표준도메인 생성 | `.claude/rules/공공표준도메인생성.md` |
| 표준코드 생성 (`_CD` 한정) | `.claude/rules/공공표준코드생성.md` |

---

## 입력

- **입력 파일**: `{요구사항번호}-db-modeling.md` (DA agent 산출). 테이블·컬럼 목록(한글 컬럼명/속성명, 추정 타입·길이) 포함.
- 입력 파일에서 검토 대상 **컬럼(한글 컬럼명) 목록**을 추출한다.

## 워크플로우

### 1. 검토 절차 (공공 우선) — 표준용어 → 표준단어 → 표준도메인 → 표준코드 → 기관용어 → 기관단어 → 기관도메인 → 기관코드

각 컬럼에 대해 공공, 기관 TSV를 `Grep` 조회한다. 존재하면 그 속성을 채택(공공표준은 수정 불가)하고, 없으면 **2. 생성 절차**로 간다.

1. **표준용어 검토**: 한글 컬럼명을 `공공표준용어.tsv`에서 조회 → 영문약어명·도메인명·허용값·저장/표현형식 채택.
2. **표준단어 검토**: 구성 최소 단위 단어를 `공공표준단어.tsv`에서 조회 → 영문약어명·도메인분류명 확인.
3. **표준도메인 검토**: 용어의 도메인명(또는 형식단어)으로 `공공표준도메인.tsv`에서 조회 → 데이터타입·길이·소수점·저장/표현형식·허용값 채택.
4. **표준코드 검토**: **영문약어명이 `_CD`로 끝나는 경우에만** `공공표준코드.tsv`에서 조회 → 코드값·코드값의미 채택. `_CD`가 아니면 건너뛴다. (사전이 없으면 "코드 사전 부재" 처리)
5. **기관용어 검토**: 한글 컬럼명을 `기관표준용어.tsv`에서 조회 → 영문약어명·도메인명·허용값·저장/표현형식 채택.
6. **기관단어 검토**: 구성 최소 단위 단어를 `기관표준단어.tsv`에서 조회 → 영문약어명·도메인분류명 확인.
7. **기관도메인 검토**: 용어의 도메인명(또는 형식단어)으로 `기관표준도메인.tsv`에서 조회 → 데이터타입·길이·소수점·저장/표현형식·허용값 채택.
8. **기관코드 검토**: **영문약어명이 `_CD`로 끝나는 경우에만** `기관표준코드.tsv`에서 조회 → 코드값·코드값의미 채택. `_CD`가 아니면 건너뛴다. (사전이 없으면 "코드 사전 부재" 처리)


### 2. 생성 절차 (기관, 공공에 없을 때) — 단어 → 용어 → 도메인 → 코드

단어는 용어의 구성요소이므로 **단어 먼저 생성** 후 용어를 조립한다. 각 단계 진입 시 해당 규칙 문서 1개만 Read한다. 등록은 기관 TSV에 **스키마 순서대로 탭 구분 행 append**.

1. **표준단어 생성** — `공공표준단어생성.md` 규칙 → `KOMSA표준단어.tsv` append. (명사형 최소단위 분할, 대표단어/이음동의어/금칙어 판정, 영문명·영문약어명 ≤6자)
2. **표준용어 생성** — `공공표준용어생성.md` 규칙 → `KOMSA표준용어.tsv` append. (표준단어 조합, 마지막 단어는 형식단어, 용어 영문약어명·도메인명 정의)
3. **표준도메인 생성** — `공공표준도메인생성.md` 규칙 → `KOMSA표준도메인.tsv` append. (형식단어→도메인분류명, `분류명+타입약어+길이` 도메인명, 저장/표현형식·허용값)
4. **표준코드 생성** — **`_CD` 한정** `공공표준코드생성.md` 규칙 → `KOMSA표준코드.tsv` append(없으면 코드 스키마 헤더로 새 파일 생성). 행정표준코드 우선 확인 → 없으면 한글/영문코드명·코드값·코드값의미·관리부서명 정의. 코드값 미상 시 임의 생성 금지, `확인 필요` 표시.

> 등록(append)은 `Bash`로 TSV에 탭 구분 행을 덧붙인다. 등록 전 항상 공공·기관 TSV를 Grep 재조회하여 **중복 등록을 방지**한다. 공공표준이 있으면 기관표준을 만들지 않는다. append 후 `awk -F'\t' '{print NF}'`로 컬럼수 정합성을 확인한다.

### 3. 등록 후 xlsx 동기화 (필수)

기관은 **TSV가 원본**이고 xlsx는 파생물이다. TSV append 직후 동기화한다.

```bash
python3 scripts/sync_metadata.py            # 기관 TSV 전체 → xlsx
# 또는 특정 파일만
python3 scripts/sync_metadata.py "docs/03. metadata/기관/KOMSA표준용어.tsv"
```

- 공공은 반대 방향(xlsx→tsv)이며 `python3 scripts/build_metadata_index.py`로만 갱신한다(공공 디렉토리만 변환).

### 4. 결과 산출 및 DA agent 전달

1. 컬럼별 확정 속성을 `docs/03. metadata/result/{요구사항번호}-column-spec.md`로 작성한다(아래 형식).
2. `SendMessage`로 **DA agent**에 산출물 경로 + 핵심 결정사항(신규 기관표준, 확인 필요 항목)을 전달한다.
3. DA agent 회신(길이·코드값 조정)은 기관 TSV에만 반영하고 다시 동기화한다. 공공은 수정하지 않는다.

## 출력 형식 (`{FR-XXX}-column-spec.md`)

```markdown
# 메타데이터 표준화 결과 — FR-XXX

## 요약
| 구분 | 건수 |
|---|---:|
| 공공표준 적용 | N |
| 기관표준 신규 생성 | N |
| 확인 필요 | N |

## 컬럼 속성
| 한글 컬럼명 | 영문약어명(컬럼) | 출처(공공/기관) | 표준용어 | 도메인명 | 데이터타입 | 길이 | 소수점 | 저장형식 | 표현형식 | 허용값 | 코드여부 | 검토의견 |
|---|---|---|---|---|---|---:|---:|---|---|---|---|---|
| 가입일자 | JOIN_YMD | 공공 | 가입일자 | 연월일C8 | CHAR | 8 | - | YYYYMMDD | YYYY-MM-DD | 유효한 날짜 | N | 공공표준용어 적용 |
| 사용자성별코드 | USER_SEX_CD | 기관 | 사용자성별코드 | 코드C1 | CHAR | 1 | - | 문자1자리 | F/M | F,M | Y | _CD → 표준코드 생성 |

## 신규 생성 기관메타데이터 (append 대상 TSV)
- KOMSA표준단어.tsv: (추가 단어 목록)
- KOMSA표준용어.tsv: (추가 용어 목록)
- KOMSA표준도메인.tsv: (추가 도메인 목록)
- KOMSA표준코드.tsv: (추가 코드 목록, 코드값 미확정 시 '확인 필요')
```

## 에러 처리

- **공공메타데이터 수정 요청**: 거부. 공공은 READONLY, 차이가 필요하면 기관 TSV로 신규 등록.
- **입력 파일 부재**: 추측 분석 금지. 발신처에 `{요구사항번호}-db-modeling.md` 위치를 `SendMessage`로 요청.
- **공공 TSV 부재/구버전**: `python3 scripts/build_metadata_index.py`로 재생성. 그래도 없으면 "공공메타데이터 사전 부재" 표기 후 `확인 필요`.
- **표준 판정 불확실**: 후보 2개 이상 제시 + 검토의견. 확정 금지.
- **코드값 목록 미상**: 임의 코드 생성 금지. `확인 필요` + 관리부서 확인 요청.

## 완료 체크리스트

- [ ] 공공메타데이터(TSV·xlsx)에 Write/Edit/append 시도 0건 (READONLY 준수)

- [ ] 표준코드 검토/생성을 `_CD`로 끝나는 컬럼에만 한정했는가
- [ ] 규칙 문서를 해당 단계에서만 Read했는가 (불필요한 선로딩 없음)
- [ ] 기관 TSV append 전 공공·기관 중복 조회를 했는가
- [ ] append 행이 컬럼 스키마 순서·탭 구분과 일치하는가 (NF 검증)
- [ ] 기관 TSV append 후 `sync_metadata.py`로 xlsx를 동기화했는가
- [ ] 모든 컬럼에 출처(공공/기관)와 영문약어명·도메인이 채워졌는가
- [ ] 결과 산출물을 DA agent에 SendMessage로 전달했는가
- [ ] 불확실/미상 항목을 `확인 필요`로 표시했는가
