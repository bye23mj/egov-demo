- 핵심 범위: 공통표준단어 개요, 관리항목, 정의 절차, 대표단어 선정, 공통표준단어명/영문명/영문약어명/설명/도메인분류명/이음동의어/금칙어 기준

## 1. Codex 사용 지침

Codex가 공공표준용어 또는 표준단어 후보를 생성할 때는 아래 순서로 판단한다.

1. 입력된 한글 컬럼명, 속성명, 용어명을 최소 단위의 명사형 단어로 분할한다.
2. 분할된 단어 중 데이터 값의 저장 형식, 표현 형식, 길이 등을 식별할 수 있는 단어를 형식단어로 판단한다.
3. 표준용어는 업무단어와 형식단어의 조합으로 구성하며, 마지막 단어는 형식단어가 되도록 한다.
4. 분할된 단어에 대해 대표단어, 이음동의어, 금칙어 여부를 검토한다.
5. 대표단어는 표준국어대사전, 법령/지침, 기술/산업 표준, 다기관 사용빈도, 국립국어원 다듬은 말 등을 기준으로 선정한다.
6. 공통표준단어영문명은 단어설명에 부합하는 영문 전체명을 선정한다.
7. 공통표준단어영문약어명은 최대 6자리 이하, 가급적 4자리 이내로 작성한다.
8. 영문약어명은 유일성을 유지하고, 개발자가 읽기 쉬운 형태로 작성한다.
9. 한글 발음을 단순 영문 표기한 약어는 원칙적으로 사용하지 않는다. 다만 순우리말이거나 적절한 영문 변환어가 없는 경우 예외적으로 허용한다.
10. 동음이의어와 금칙어는 원칙적으로 사용하지 않는다. 필요 시 복합어 또는 다른 대표단어로 대체한다.

## 2. Codex 출력 권장 형식

Codex는 표준용어 또는 표준단어 후보 생성 시 다음 표 형식을 우선 사용한다.

```markdown
| 구분 | 입력값 | 분할 단어 | 형식단어 | 표준단어 후보 | 영문명 | 영문약어명 | 도메인분류명 | 이음동의어 | 금칙어 | 검토의견 |
|---|---|---|---|---|---|---|---|---|---|---|
| 표준단어 | 사용자 | 사용자 | N | 사용자 | User | USER | - | 사용인 | 유저 | 표준국어대사전 등재어를 대표단어로 사용 |
| 표준용어 | 사용자등록일자 | 사용자 / 등록 / 일자 | 일자 | 사용자등록일자 | User Registration Date | USER_REG_YMD | 연월일 | - | - | 마지막 단어가 형식단어이므로 용어로 사용 가능 |
```

| 공통표준단어 영문명 | 공통표준단어에 대한 영문 전체명 | 설명에 부합하는 쉬운 영문명 우선 |
| 공통표준단어 영문약어명 | 공통표준단어의 영문 약어 표기 | 최대 6자리 이하, 가급적 4자리 이내 |

| 이음동의어 목록 | 대표단어와 의미가 같은 단어 | 대표단어 사용을 우선하되 필요 시 허용 |
| 금칙어 목록 | 사용 불가 단어 | 약어, 비표준어, 잘못된 외래어, 비하어 등 배제 |

## 4. 공통표준단어 정의 절차

### 4.1 단어 분할

- 표준화하고자 하는 용어를 용어 구성의 최소 단위인 단어로 분할한다.
- 단어는 자립적인 의미를 가지는 최소 단위로 분할한다.
- 예: `이메일수신동의여부` -> `이메일 / 수신 / 동의 / 여부`

### 4.2 형식단어 도출

- 분할된 단어 중 데이터 값의 저장/표현 형식, 길이 등 특정 형식을 식별할 수 있는 단어를 형식단어로 도출한다.
- 형식단어는 일반적으로 용어의 마지막에 위치한다.
- 예: `가입일자` -> 형식단어 `일자`
- 예: `개인정보수집동의여부` -> 형식단어 `여부`
- 마지막 단어가 형식단어가 아니면 표준용어로 사용하기 어렵다.

### 4.3 대표단어 선정

대표단어는 아래 기준을 종합하여 선정한다.

| 우선 기준 | 판단 기준 | 예시 |
|---|---|---|
| 표준어 | 국립국어원 표준국어대사전에 등재된 단어 | 사용자 O, 유저 X |
| 기술/산업 표준 | KS, TTA 등 기술표준에서 통용되는 용어 | URL, SMS, IP |
| 법령/지침 | 법령, 지침에서 정의한 단어 | 개인정보, 법인등록번호 |
| 다기관 사용빈도 | 중앙 메타데이터 관리시스템 등에서 사용 기관 수가 많은 단어 | 사용자, 소유자 등 |
| 순화어 | 국립국어원 다듬은 말을 반영 | 언택트 -> 비대면, 구좌 -> 계좌 |

## 5. 공통표준단어명 정의 기준

Codex는 공통표준단어명 생성 시 다음 기준을 적용한다.

1. 사용문자는 한글, 영문, 숫자로 구성하며 영문자는 대문자만 사용한다.
2. 한글 문자수는 제한하지 않지만 한 글자 단어는 의미 혼동 가능성이 있으므로 가급적 구체적 단어로 정의한다.
3. 단어는 명사형으로 정의하고 동사, 접속사, 복수표시, 소유격 형태는 사용하지 않는다.
4. 단어는 띄어쓰기를 하지 않는다.
5. 특수문자 `/`, `_`, `-`, `+`, `(`, `)`는 사용하지 않는다.
6. 순화대상 단어와 순화된 단어가 함께 사용되는 경우 가급적 순화된 단어를 사용한다.
7. 기관명 또는 긴 단어 등은 축약하지 않는다.
8. 한 글자 축약어는 혼동 우려가 있으므로 가급적 풀어쓴 단어를 사용한다.
9. 동음이의어는 원칙적으로 허용하지 않고, 다른 단어 또는 복합어로 대체한다.
10. 숫자와 조합된 단어는 의미가 분명하도록 정의하고 필요 시 수식어를 추가한다.
11. 두음법칙이 발생하는 단어는 기준 단어를 표준단어로 정의하고 나머지는 이음동의어로 정의한다.
12. 접두사는 단일어 앞에 붙여 복합어로 정의할 수 있다.
13. 접미사는 단일어 뒤에 붙여 복합어로 정의할 수 있다.
14. 단일어와 단일어가 결합되어 새로운 의미를 갖는 경우 복합어로 등록한다.
15. 단어 결합 시 영문명의 의미가 모호하거나 달라지는 경우 복합단어로 등록하고 새 영문명/약어를 정의한다.
16. 표준국어대사전에 외래어가 한글로 등재된 경우 한글표기를 사용한다.
17. 널리 알려진 영문약어가 표준국어대사전에 외래어로 등재되지 않은 경우 영문약어를 그대로 사용할 수 있다.
18. 외래어를 한글단어로 번역할 수 있는 경우 의역된 한글 단어를 사용한다.

## 6. 공통표준단어 영문명 정의 기준

1. 해당 단어가 이음동의어인지 먼저 확인한다.
2. 단어 설명에 부합하는 영문명이 있는지 확인한다.
3. 동일 영문명이 중복되는 경우 단어 사용 빈도를 조사한다.
4. 사용 빈도가 높은 단어에 쉬운 영문단어를 배정한다.
5. 영문명 중복을 회피할 수 없으면 영문약어명을 다르게 생성한다.

예시:

| 공통표준단어명 | 후보영문명 | 선정 영문명 | 검토의견 |
|---|---|---|---|
| 갱신 | Renewal | Update | Renewal은 개편 의미도 포함하므로 Update 선택 |
| 지하 | Basement | Underground | Basement는 지하실 의미가 강하므로 Underground 선택 |
| 출생 | Born | Birth | Born은 동사형 의미가 강하므로 Birth 선택 |

## 7. 공통표준단어 영문약어명 정의 기준

1. 국내외 표준으로 사용하는 영문약어는 해당 약어를 준용한다.
2. IT 용어, 경제 용어, 고유명사는 해당 영문약어를 준용한다.
3. 길이는 최대 6자리 이하, 가급적 4자리 이내로 정의한다.
4. 영문명의 첫 번째 로마자 이외의 모음을 제거하고 각 음절의 대표자음을 사용하여 구성한다.
5. 두 개의 자음이 연속할 경우 가급적 한 개의 자음은 생략한다.
6. 첫 자가 모음이면 첫 자를 그대로 유지한다.
7. 숫자가 반드시 필요한 경우 가급적 영문화하거나 영문 뒤에 위치한다.
8. 영문명이 4자 이내이면 영문명을 약어로 그대로 사용할 수 있다.
9. 한글 발음을 영문으로 표기한 단어는 원칙적으로 사용하지 않는다.
10. 일반적이거나 자주 사용되는 영문약어는 관용어 기준을 적용한다.
11. 가독성을 높이기 위해 영문 단어 앞자리 축약, 첫 글자와 마지막 글자 조합, 특징 있는 자음 조합을 사용할 수 있다.
12. 영문약어는 유일성을 유지하는 것이 원칙이다. 다만 동일 영문명을 쓰는 것이 타당한 이음동의어는 예외적으로 허용할 수 있다.
13. 영문 정식명에 조사나 전치사 등이 있는 경우 생략하여 약어를 구성할 수 있다.

예시:

| 단어 | 영문명 | 약어 예시 |
|---|---|---|
| 수요일 | Wednesday | WED |
| HTML | HTML | HTML |
| 변경 | Change | CHG |
| 요청 | Demand | DMND |
| 입력 | Input | INPT |
| 성공 | Success | SCS |
| 조치 | Action | ACTN |
| 가입 | Join | JOIN |
| 오류 | Error | ERR |
| 사업 | Business | BIZ |
| 삭제 | Deletion | DEL |

## 9. 이음동의어 정의 기준

1. 표준국어대사전에서 정의가 동일한 경우 이음동의어로 정의할 수 있다.
2. 의미 차이가 있으면 원칙적으로 이음동의어로 등록하지 않는다. 다만 사용상 차이가 없으면 이음동의어로 정의할 수 있다.
3. 줄임말이거나 맞춤법상 변형되는 단어도 이음동의어로 정의할 수 있다.
4. 영어와 한글 전환 시 변형되는 단어도 이음동의어로 정의할 수 있다.
5. 복합어가 표준단어와 의미상 동일하고 실제 DB에서 사용되는 경우 이음동의어로 정의할 수 있다.

대표단어 사용이 원칙이나, 운영시스템과 충돌하는 경우 이음동의어를 사용할 수 있다. 이때 영문약어명과 표준분류명은 대표단어와 동일하게 사용한다.

---

## 10. 금칙어 정의 기준

1. 법령에서 사용이 허락되지 않거나 특정 시점 이후 사용이 중지된 단어는 금칙어로 정의한다.
2. 표준단어의 의미를 풀어서 설명한 형태로 사용되는 단어는 금칙어로 정할 수 있다.
3. 영어를 한글로 표현할 때 외래어 표기법상 잘못된 경우 금칙어로 정할 수 있다.
4. 표준단어와 동일한 의미의 영어를 한글로 표현했으나 표준국어대사전에 등재되지 않은 경우 금칙어로 정할 수 있다.
5. 비하하거나 하대하는 뜻이 있는 단어는 금칙어로 정할 수 있다.
6. 긴 단어를 축약해서 사용하는 약어는 금칙어로 정할 수 있다.

## 11. Codex 프롬프트 예시

```text
너는 공공데이터베이스 표준화 관리 매뉴얼의 공통표준단어 정의 기준을 따르는 데이터 표준화 도우미다.

입력된 한글 컬럼명 또는 업무용어를 다음 기준으로 분석하라.

1. 최소 단위 명사형 단어로 분할한다.
2. 마지막 단어가 형식단어인지 확인한다.
3. 대표단어, 이음동의어, 금칙어 후보를 검토한다.
4. 공통표준단어명은 한글/영문/숫자만 사용하고 띄어쓰기와 특수문자를 제거한다.
5. 영문명은 단어 설명에 맞는 쉬운 영문 전체명으로 정의한다.
6. 영문약어명은 최대 6자리 이하, 가급적 4자리 이내로 생성한다.
7. 한글 발음을 영문으로 단순 표기하지 않는다.
8. 동음이의어와 금칙어는 원칙적으로 사용하지 않고 대체 단어를 제안한다.
9. 결과는 표준단어 후보, 표준용어 후보, 도메인분류명, 검토의견으로 구분해서 출력한다.

출력 형식:
| 입력값 | 분할 단어 | 형식단어 | 표준단어 후보 | 영문명 | 영문약어명 | 도메인분류명 | 이음동의어 | 금칙어 | 검토의견 |
```
