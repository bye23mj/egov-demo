# 공공표준용어 생성 참고자료 
- 용도: Codex 또는 Claude Code에서 공공표준용어, 공통표준단어, 영문명, 영문약어명 생성 시 참조
---
## Codex 사용 지침

이 문서는 공공표준용어 생성 시 다음 기준을 우선 적용하기 위한 참고자료이다.

1. 공통표준용어 또는 기관/DB 표준용어를 생성할 때는 표준단어와 표준도메인의 조합을 고려한다.
2. 표준단어는 명사형 최소 단위로 분할한다.
3. 용어의 마지막 단어가 데이터 형식을 식별할 수 있는 경우 형식단어로 판단한다.
4. 대표단어를 선정할 때는 표준국어대사전, 법령/지침, 기술/산업표준, 기관 사용빈도, 국립국어원 다듬은 말 등을 고려한다.
5. 동음이의어는 원칙적으로 허용하지 않고, 필요 시 복합어로 대체하거나 단어설명과 영문명을 구분한다.
6. 영문명은 단어 설명에 부합하는 Full Name으로 정의한다.
7. 영문약어명은 데이터베이스 컬럼명으로 사용될 수 있으므로 유일성, 가독성, 간결성을 확보한다.
8. 이음동의어와 금칙어를 함께 관리하여 표준단어 선택 기준을 명확히 한다.

### 1. 이음동의어
해당 공통표준용어를 구성하는 공통표준단어에 이음동의어가 존재하고 이음동의어가 포함된 용어를 사용하는 경우, 용어 이음동의어 목록을 작성한다.

#### 표 IV-12. 용어 이음동의어 목록 작성 예시

| 표준용어명 | 용어 이음동의어 목록 |
|---|---|
| 이메일주소 | 전자우편주소 |
| 저자명 | 저자이름, 글쓴이명, 글쓴이이름, 지은이명, 지은이이름 |
| 아파트명 | 아파트이름, 아파트명칭 |

### 2. 행정표준코드명

> 참고: 행정표준코드에 대한 세부사항은 행정표준코드시스템을 참조한다.

## 3. 공통표준단어
### 3.1 공통표준단어 개요
- 편의나 관습 또는 기관 업무특성에 따라 다르게 사용하는 단어들을 대표성을 지닌 단어로 표준화한다.
- 범정부 차원에서 일관성 있는 의미와 명칭의 단어를 생성하고 활용한다.
- 공통표준단어 중 형식단어, 즉 분류어를 활용하여 데이터 형식을 일관된 기준으로 지정한다.
- 동일한 형식의 공통표준용어를 생성하고 관리할 수 있다.

### 3.2 공통표준단어 관리항목
공통표준단어는 다음 항목으로 구성한다.
- 공통표준단어명
- 공통표준단어영문명
- 공통표준단어영문약어명
- 공통표준단어설명
- 형식단어 여부
- 공통표준도메인 분류명
- 이음동의어 목록
- 금칙어 목록

#### 표 IV-13. 공통표준단어 관리항목 및 예시
| 공통표준단어명 | 공통표준단어 영문약어명 | 공통표준단어 영문명 | 공통표준단어 설명 | 형식단어 여부 | 공통표준도메인 분류명 | 이음동의어 목록 | 금칙어 목록 |
|---|---|---|---|---|---|---|---|
| 파일 | FILE | File | ... | N | - | ... | 화일, FILE |
| 구매 | PRCHS | Purchase | ... | N | - | 구입, 매입 | - |
| 생년월일 | BRDT | Birth Date | ... | Y | 연월일 | - | - |

### 3.3 공통표준단어 정의 기준

### 3.3.1 공통표준단어 정의 절차 및 기준

공통표준단어 정의는 다음 3단계 절차로 수행한다.

#### 그림 IV-4. 공통표준단어 정의절차

| 단계 | 주의 사항 |
|---|---|
| 1. 단어분할 | 명사인 최소단위의 단어로 분할 |
| 2. 형식단어 도출 | 형식 식별이 가능한 단어로, 도메인분류명과 함께 형식단어, 즉 분류어로 지정 |
| 3. 대표단어 선정 | 분할된 단어를 기준으로 단어 의미와 사용 빈도수를 고려하여 대표성을 지니는 단어로 선정 |

---
#### 3.3.1.1. 단어 분할

표준화하고자 하는 용어들을 대상으로 용어 구성의 최소단위인 단어로 분할한다.

공통표준단어는 기관의 데이터베이스에서 사용 빈도가 높은 용어를 표준용어 후보군으로 선정하여 표준화를 수행한 후, 용어에서 단어를 분할하는 순서로 진행한다.

예시:

```text
이메일수신동의여부 -> 이메일 / 수신 / 동의 / 여부
```

분할 기준은 자립적인 의미를 가지는 최소단위이다.

#### 3.3.1.2. 형식단어 도출

분할된 단어들 중 형식 식별이 가능한 형식단어를 도출한다.

형식단어는 데이터값의 저장 형식, 표현 형식, 길이 등 특정한 형식을 식별할 수 있는 단어이다. 용어 구성 시 일반적으로 가장 뒤에 위치한다.

#### 형식단어 도출 예시

| 한글 컬럼명 | 단어1 | 단어2 | 단어3 | 단어4 | 형식단어 여부 | 형식단어 |
|---|---|---|---|---|---|---|
| 가입일자 | 가입 | 일자 | - | - | Y | 일자 |
| 개인정보수집동의여부 | 개인정보 | 수집 | 동의 | 여부 | Y | 여부 |
| 이메일수신 | 이메일 | 수신 | - | - | N | - |

주의:

- `이메일수신`은 마지막 단어가 `수신`으로 형식단어가 아니므로 용어로 사용할 수 없다.

#### 3.3.1.3. 대표단어 선정

단어 분할 후 의미가 유사한 단어들을 식별하고 그룹핑한 다음, 그 가운데 대표단어를 선정한다. 대표단어 이외의 단어들은 이음동의어 또는 금칙어로 지정할 수 있다.

#### 표 IV-15. 대표단어 선정 기준, 일부

| 절차 | 대표단어 선정 기준 | 기준 설명 | 예시 |
|---|---|---|---|
| 1 | 국립국어원에서 규정한 표준어 | 국립국어원에서 표준어로 인정하여 표준국어대사전에 등재된 단어. 표준국어대사전에 위배되는 명확한 비표준어는 금칙어로 지정 | 사용자(O) / 유저(X), 소유자(O) / 오너(X) |
| 2 | 기술/산업표준에서 통용되는 단어 | 표준어로 인정되지는 않았으나 기술/산업표준 등에서 통용되는 영문약어 등의 단어 | URL, SMS, IP 등 |

### 3.3.2 공통표준단어 관리항목 정의 기준

대표단어로 선정된 단어를 공통표준단어명으로 정의하고 공통표준단어영문명, 공통표준단어설명 등 세부적인 관리항목을 정의한다.

#### 3.3.2.1. 공통표준단어명
표준단어는 최소 단위의 명사형 업무단어로 정의한다.
단어는 다음과 같이 구분한다.
- 단일어
- 복합어
- 합성어
- 파생어

단일어를 결합하여 새로운 의미를 가지거나 관용적으로 복합어 형태로 사용하는 빈도가 높은 경우는 복합어를 생성하여 사용한다.

#### 표 IV-15. 대표단어 선정 기준, 계속

| 절차 | 대표단어 선정 기준 | 기준 설명 | 예시 |
|---|---|---|---|
| 3 | 법령/지침에서 정의하는 단어 | 법령, 지침 등에서 정의하고 있는 단어. 복합어 포함. 여러 단어가 결합된 복합어 형태도 법령 및 지침에서 정의된 경우 등록 | 개인정보, 법인등록번호 등 |
| 4 | 다기관 사용빈도 높은 단어 | 단어 중 사용하는 기관수가 많은 단어를 대표단어로 선정 | 사용기관 수 및 사용빈도는 중앙 메타데이터 관리시스템에 등록된 컬럼 정보 기준 |
| 5 | 국립국어원 다듬은 말 | 국립국어원에서 제공하는 다듬은 말을 참조하여 공공성이 높은 단어를 대표단어로 선정 | 언택트 -> 비대면, 구좌 -> 계좌 |

#### 다기관 사용빈도 기준 예시

| 단어명 | 사용기관 수 | 선정 결과 |
|---|---:|---|
| 소유자 | 12개 | 대표단어 |
| 소유주 | 8개 | 이음동의어 |
| 소유인 | 4개 | - |
| 사용자 | 24개 | 대표단어 |
| 사용인 | 7개 | 이음동의어 |

#### 표 IV-16. 공통표준단어명 정의 기준, 일부

| 번호 | 정의 기준 | 예시 |
|---:|---|---|
| 1 | 사용문자는 한글, 영문 및 숫자로 구성하며 영문자는 대문자만 사용한다. | Html(X) -> HTML(O), sms(X) -> SMS(O) |
| 2 | 한글 문자수는 제한을 두지 않는다. 단, 한 글자의 단어는 다양한 의미를 가질 수 있으므로 가급적 사용을 최소화하고 구체적인 단어로 정의한다. | 남 -> 남자, 여 -> 여자, 집 -> 자택/주택 |
| 3 | 단어는 명사형으로 정의하고 동사, 접속사, 복수표시, 소유격 형태는 사용하지 않는다. | 신청한사람(X) -> 신청자(O), 개인에관한정보(X) -> 개인정보(O), 공급받는자(O) |
| 4 | 단어는 띄어쓰기를 하지 않는다. | 주민 등록 번호(X) -> 주민등록번호(O) |
| 5 | 특수문자 `/`, `_`, `-`, `+`, `(`, `)`는 사용하지 않는다. | 내_외국(X) -> 내외국(O), 전화/번호(X) -> 전화번호(O) |
| 6 | 순화대상 단어와 순화된 단어를 함께 사용하고 있는 경우 가급적 순화된 단어를 사용한다. | 리플 -> 댓글, 계출 -> 신고 |

---

#### 표 IV-16. 공통표준단어명 정의 기준, 계속

| 번호 | 정의 기준 | 예시 |
|---:|---|---|
| 7 | 기관명 또는 긴 단어 등은 축약해서 사용하지 않는다. 약어 사용을 금지한다. | 전번(X) -> 전화번호(O), 주민번호(X) -> 주민등록번호(O) |
| 8 | 한 글자의 축약어는 다른 단어와 붙여 쓸 경우 혼동될 우려가 있으므로 가급적 풀어쓴 단어를 사용한다. | 가(假) -> 임시(O), 외(外) -> 제외(O) |
| 9 | 범정부 차원에서는 원칙적으로 동음이의어 사용을 허용하지 않고 다른 단어로 대체하거나, 사용 빈도가 낮은 단어를 복합어로 대체한다. | 다리(leg) -> 다리, 다리(bridge) -> 교량, 수입(Income) -> 수입, 수입(Import) -> 수입관세 |
| 10 | 숫자와 조합된 단어는 의미가 불분명하지 않도록 정의하며 필요한 경우 수식어를 추가하여 의미를 명확히 한다. | 감염1(X) -> 일차감염 |
| 11 | 두 개의 단어가 두음법칙이 발생하는 경우 기준이 되는 단어를 표준단어로 정의하고, 나머지는 이음동의어로 정의한다. | 연도/년도 -> 연도(표준단어), 년도(이음동의어). 연월/년월 -> 연월(표준단어), 년월(이음동의어) |

동음이의어 처리 참고:

- 대체방안 1: 사용 빈도가 낮은 단어를 다른 한글단어로 대체하여 공통표준단어로 등록한다.
- 대체방안 2: 단어를 반드시 사용해야 하는 경우 복합어를 공통표준단어로 등록하여 사용한다.
- 개별 기관 차원에서도 원칙적으로 동음이의어 사용을 허용하지 않는다.
- 부득이한 경우 단어설명과 단어영문명을 구분하여 기관 표준단어로 정의할 수 있다.

#### 표 IV-16. 공통표준단어명 정의 기준, 계속

| 번호 | 정의 기준 | 예시 |
|---:|---|---|
| 12 | 복합어 - 파생어: 접두사는 단일어 앞에 붙여 복합어로 정의할 수 있다. | 비 + 과세 -> 비과세, 부 + 적합 -> 부적합 |
| 13 | 복합어 - 파생어: 접미사는 단일어 뒤에 붙여 복합어로 정의할 수 있다. | 관리자, 대표자, 신청자, 외국인 |
| 14 | 복합어 - 합성어: 단일어 + 단일어로 구성된 복합어가 새로운 의미를 가진 경우, 정확한 의미를 인식하도록 복합어로 등록한다. | 전화 + 번호 -> 전화번호, 개인 + 정보 -> 개인정보, 여권 + 번호 -> 여권번호 |
| 15 | 복합어 - 합성어: 단어 결합 시 영문명의 의미가 모호하거나 달라지는 경우, 복합단어로 등록하고 적합한 새로운 단어영문명 및 약어를 정의한다. | 세금계산서 -> Tax Invoice, 대차대조표 -> Balance Sheet / BS |
| 16 | 외래어: 외래어 중 표준국어대사전에 한글로 등재된 경우, 한글표기를 사용한다. 이 경우 영문표기는 사용하지 않는다. | 로그인(O) -> Log-in(X), 이메일(O) -> Email(X) |
| 17 | 영문약어: 관용적으로 널리 알려진 영문약어 중 표준국어대사전에 한글 외래어로 등재되지 않은 경우, 영문약어를 그대로 사용한다. | HTML, SMS |

접두사 예시:

```text
부, 재, 미, 비, 선, 신, 구, 전, 후, 모, 자, 유, 무, 별, 총, 원, 대, 중, 소, 주, 부 등
```

접미사 예시:

```text
용, 사, 처, 자, 인, 증, 지, 료, 서, 장, 금, 비, 권, 성, 소, 형, 표, 식, 화, 도 등
```

#### 3.3.2.2. 공통표준단어영문명

- 공통표준단어영문명은 공통표준단어명을 영문으로 표기한 단어영문명, 즉 Full Name을 의미한다. 공통표준단어 설명에 부합하는 단어영문명을 정의해야 한다.
- 공통표준단어영문명은 공통표준단어영문약어명을 만드는 기준이 된다.
- 공통표준단어영문명은 분야별 전문용어사전, 정보통신용어사전, 한국산업표준 KS 표준용어, 법령표준용어 등을 참조하며, 이외에 출판사 영어사전 등을 활용한다.
- 공통표준단어에 해당하는 영문명 선정은 공통표준단어 설명에 부합하는 영어 단어를 선정한 후, 단어에 해당하는 영문명이 중복되는 경우 공통표준단어 영문명 정의 기준에 따라 선정한다.

#### 표 IV-17. 공통표준단어 영문명 정의 기준

| 순서 | 정의 기준 | 예시 |
|---:|---|---|
| 1 | 단어가 이음동의어는 아닌지 확인한다. | 데이터(Data), 자료(Data) |
| 2 | 설명에 부합되는 영문명은 없는지 다시 확인한다. | 계정(Account), 계좌(Account -> Bank Account), 품목(Item), 물품(Item -> Commodity) |
| 3 | 중복되는 영문명에 해당하는 단어의 사용 빈도를 조사한다. | 건물 45개 기관 106회, 건축물 40개 기관 32회 |
| 4 | 사용 빈도가 높은 단어에 쉬운 영문단어를 배정한다. | 건물(Building), 건축물(Building -> Buildings & Structures) |
| 5 | 영문명 중복을 회피할 수 없는 경우 영문약어명을 다르게 생성해야 한다. | 단체(Group/GRP), 그룹(Group/GROUP), 문서(Document/DOC), 서류(Document/DCMNT) |

#### 3.3.2.3. 공통표준단어영문약어명

공통표준단어영문약어명은 공통표준단어영문명에 대한 약어로, 공통표준용어에 대한 영문명 생성 시 사용된다.

궁극적으로는 데이터베이스 구축 시 적용되므로 다음 요건을 충족해야 한다.

- 중복 방지를 위해 유일성이 있어야 한다.
- 시스템 개발자에게 가독성이 높게 정의되어야 한다.

| 2 | IT 용어, 경제 용어, 고유 명사는 해당 영문약어를 준용한다. | HTML, SMS |
| 3 | 길이는 최대 6자리 이하로 정의하며, 가급적 4자리 이내로 정의한다. | - |

# 공공표준용어 생성용 요약 규칙

Codex가 공공표준용어 또는 표준단어 후보를 생성할 때 다음 규칙을 적용한다.

## 1. 용어 분할 규칙

```text
입력 용어 -> 자립적 의미를 가지는 명사형 최소 단위로 분할
```

예시:

```text
이메일수신동의여부 -> 이메일 / 수신 / 동의 / 여부
가입일자 -> 가입 / 일자
개인정보수집동의여부 -> 개인정보 / 수집 / 동의 / 여부
```

## 2. 형식단어 판단 규칙

- 마지막 단어가 데이터 형식, 저장 형식, 표현 형식, 길이 등을 식별할 수 있으면 형식단어로 본다.
- 형식단어는 공통표준도메인 분류명과 연결한다.
- 마지막 단어가 형식단어가 아니면 표준용어로 사용하기 어려우므로 재검토한다.

## 3. 대표단어 선정 우선순위

1. 국립국어원 표준국어대사전에 등재된 표준어
2. 기술/산업표준에서 통용되는 단어 또는 영문약어
3. 법령/지침에서 정의된 단어
4. 여러 기관에서 사용 빈도가 높은 단어
5. 국립국어원 다듬은 말 등 공공성이 높은 순화어

## 4. 공통표준단어명 금지/주의사항

- 동사형, 접속사, 복수표시, 소유격 형태를 사용하지 않는다.
- 띄어쓰기를 사용하지 않는다.
- 특수문자를 사용하지 않는다.
- 긴 단어와 기관명은 축약하지 않는다.
- 한 글자 축약어는 가급적 풀어 쓴다.
- 동음이의어는 원칙적으로 허용하지 않는다.
- 표준국어대사전에 한글 외래어로 등재된 경우 한글 표기를 사용한다.
- 널리 통용되고 한글 외래어로 등재되지 않은 영문약어는 그대로 사용할 수 있다.

## 5. 영문명 생성 규칙

- 공통표준단어 설명에 부합하는 Full Name을 선정한다.
- 분야별 전문용어사전, 정보통신용어사전, KS 표준용어, 법령표준용어 등을 우선 참조한다.
- 동일한 영문명이 여러 한글 단어에 대응되면 이음동의어 여부, 의미 적합성, 사용 빈도, 쉬운 영문 배정 가능성을 검토한다.
- 영문명 중복을 피할 수 없으면 영문약어명을 다르게 생성한다.

## 6. 영문약어명 생성 규칙

- 국내외 표준 약어가 있으면 해당 약어를 준용한다.
- IT 용어, 경제 용어, 고유 명사는 통용되는 약어를 준용한다.
- 길이는 최대 6자리 이하, 가급적 4자리 이내로 생성한다.
- 데이터베이스 컬럼명으로 사용될 수 있으므로 유일성과 가독성을 확보한다.
