# ADR-0002 — 선박기본정보 반복그룹 1:N 정규화

- 상태: 채택(Accepted)
- 일자: 2026-06-25
- 관련: ADR-0001(적재형 단일테이블)의 후속 — 보완항목 "반복그룹 비정규화" 해소

## 맥락
TB_VSL_BSC는 원천 1:1 적재를 위해 반복그룹(주기관×2, 차기검사×5, 여객등급4)을 평면 컬럼으로 보유한다.
이는 확장성(주기관 3대 이상 불가)·질의성(검사 이력 집계 곤란)·정합성(빈 슬롯 NULL)에 한계가 있다.

## 결정
반복그룹을 부모-자식 1:N으로 정규화한다.
1. **TB_VSL_MENG**(선박주기관) — 주기관 N건, UK(VSL_BSC_ID, MENG_SEQ)
2. **TB_VSL_NXTM_INSP**(선박차기검사) — 차기검사 N건, UK(VSL_BSC_ID, NXTM_INSP_SEQ)
3. **TB_VSL_PSNR_CPCT**(선박여객정원) — 여객등급 정원 N건, UK(VSL_BSC_ID, PSNR_GRD_SE)
- 각 자식: 인조키 PK + FK(VSL_BSC_ID→TB_VSL_BSC) + FK 인덱스.
- 데이터 분해는 INSERT...SELECT로 **값 존재 슬롯만** 적재(NULL/0 제외).

## 사유
- 가변 개수 수용, 검사이력·정원 집계 질의 단순화, NULL 슬롯 제거로 정합성 향상.
- 부모는 요약(여객소계 등) 유지로 단건 조회 성능 보존.

## 결과/후속
- 마이그레이션: `ddl/03_normalize_split.sql` (자식 생성 + 데이터 분해).
- 부모의 반복그룹 컬럼 DROP은 **검증 후 수동 실행**(스크립트에 주석 처리).
- 적용은 dba-agent로 수행(본 ADR은 설계·마이그레이션 정의까지).
- 범주형 표준코드화(여객등급구분/검사종류 등)는 별도 후속 과제로 유지.
