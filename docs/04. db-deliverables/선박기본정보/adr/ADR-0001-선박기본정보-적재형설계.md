# ADR-0001 — 선박기본정보 적재형 단일테이블 설계

- 상태: 채택(Accepted)
- 일자: 2026-06-25
- 대상: TB_VSL_BSC (komsa)

## 맥락
원천 `work/선박기본정보.csv`(45컬럼, 68,933행)를 Oracle에 적재한다. 데이터에는
(1) 자연키 후보 `선박번호`에 중복 22건, (2) 주기관(×2)·차기검사(×5)·여객등급 반복그룹이 존재한다.

## 결정
1. **인조키 채택**: 주식별자를 SEQUENCE 인조키 `VSL_BSC_ID`로 둔다. `선박번호`는 중복 허용 비유일 인덱스(IX_TB_VSL_BSC_VSL_NO).
2. **적재형 단일테이블**: 반복그룹을 1:N 분리하지 않고 원천 1:1 평면 구조로 적재한다(스테이징/기본정보 목적).
3. **원천값 보존**: 범주형 컬럼은 표준코드로 치환하지 않고 원천 문자값을 유지한다.

## 사유
- 적재 단계 목표는 원천 충실 재현·추적성 확보. 정규화/코드화는 활용 단계 과제로 분리.
- 중복 선박번호를 UK로 강제하면 적재 실패 → 인조키로 회피하고 품질대장에 중복을 관찰항목으로 등록.

## 결과/후속
- 정규화(주기관·차기검사·여객정원 1:N)와 코드 표준화는 후속 ADR로 분리한다.
- 적재 추적용 `LOAD_DT`(DEFAULT SYSTIMESTAMP) 컬럼을 추가한다.
