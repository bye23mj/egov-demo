# 데이터구조 검증 결과 (정규화) — 선박기본정보

- 대상: Oracle 11g / komsa / 부모 TB_VSL_BSC + 자식 TB_VSL_MENG·TB_VSL_NXTM_INSP·TB_VSL_PSNR_CPCT
- 입력: 선박기본정보-정규화-db-modeling.md + 표준검토(정원/PSCP 등록)

## 검증 항목 판정
| 범주 | 점검 | 판정 | 사유 |
|---|---|---|---|
| 정규화 | 반복그룹 제거(주기관·차기검사·여객등급) | **적합** | 1:N 분리로 ADR-0001 보완 해소. 부모는 요약(여객소계) 유지 |
| 식별자 | 자식 인조키 + 복합 유일키 | **적합** | 자식 PK=SEQ 인조키, (VSL_BSC_ID+순번/구분) UK로 중복 차단 |
| 참조무결성 | FK ↔ 부모 PK 도메인 일치 | **적합** | 자식 VSL_BSC_ID NUMBER ↔ 부모 PK NUMBER 일치. 관계 1:N |
| 도메인 | 타입·길이·NULL | **적합** | 순번 NUMBER(2), 등급구분 VARCHAR2(10), 일시 DATE 표준도메인 정합 |
| 코드 | 여객등급구분(특/1/2/3)·주기관종류·검사종류 | **보완** | 원천값 보존, 표준코드화는 후속(분포는 분석노트 참조) |
| 명명 | 표준약어 기반 | **적합** | MENG/NXTM/INSP/PSNR/PSCP/SEQ/SE/GRD 표준 적용 |
| 테이블명 | 명명규칙 준수 | **적합** | TB_VSL_MENG/NXTM_INSP/PSNR_CPCT, 금지명 회피, 엔터티 정합 |
| 제약/인덱스 | PK/FK/UK/인덱스 | **적합** | 각 자식 PK·FK 인덱스, FK 컬럼 인덱스 권장 |

## DBMS별(Oracle 11g)
| 항목 | 점검 | 판정 |
|---|---|---|
| 식별자 길이(30byte) | 최장 VSL_NXTM_INSP_ID(16) | 적합 |
| 채번 | 자식별 SEQUENCE 3종 | 적합 |
| FK 인덱스 | FK 컬럼 비유일 인덱스 생성 | 적합(DDL 반영) |

## 종합 판정
- **적합** — 정규화 모델 산출물·마이그레이션 진행 가능.
- ADR-0001의 보완항목 "반복그룹 비정규화" → **해소(ADR-0002로 기록)**.
- 잔여 보완: 범주형 표준코드화(후속 과제 유지).
