# 데이터구조 검증 결과 — 선박기본정보

- 대상 DBMS: Oracle 11g XE / 대상 테이블: **TB_VSL_BSC** (komsa)
- 입력: 선박기본정보-db-modeling.md + 표준검토결과(KOMSA 단어 5건 등록 반영)

## 검증 항목 판정

| 범주 | 점검 내용 | 판정 | 사유/조치 |
|---|---|---|---|
| 정규화 | 주기관(×2)·차기검사(×5)·여객등급 **반복그룹** 존재 | **보완** | 3NF상 1:N 분리 권장. 단 **원천 1:1 적재 목적**으로 평면 단일테이블 채택(의도적). 향후 정규화는 ADR 기록 |
| 식별자 | 자연키 `선박번호` 유일성 | **적합(인조키 채택)** | 선박번호 중복 22건 → 유일성 미충족 확인. **인조키 VSL_BSC_ID(SEQUENCE)** 주식별자, 선박번호는 비유일 인덱스 |
| 참조무결성 | FK ↔ PK 도메인 일치 | **해당없음** | 단일 테이블, 외부 참조 없음 |
| 도메인 | 타입·길이·NULL·기본값 표준도메인 정합 | **적합** | 숫자(NUMBER)·일자(DATE)·명칭(VARCHAR2) 표준도메인 매핑. 결측 다수로 NULL 허용(VSL_BSC_ID만 NOT NULL) |
| 코드 | `_NM` 범주(구분/선박구분/재질/주기관종류/차기검사종류/계선여부) | **보완** | 현 적재는 원천 문자값 보존. 표준코드 매핑은 후속 과제(코드 8종 분포는 데이터분석 참조) |
| 명명 | 표준용어 영문약어명 기준 일관성 | **적합** | 공공표준단어 우선 적용, 미등록 5건(너비/마력/계선/주기관/특)은 KOMSA 기관표준 등록 후 반영 |
| 테이블명 | `.claude/rules/테이블명명규칙.md` 준수 | **적합** | `TB_VSL_BSC` = TB_[VSL 선박]_[BSC 기본]. 금지명(INFO/MASTER 등) 회피, 표준약어 기반, 엔터티 정합 |
| 제약/인덱스 | PK/UK/NOT NULL/인덱스 적정성 | **적합** | PK_TB_VSL_BSC(인조키), IX_TB_VSL_BSC_VSL_NO(비유일), SEQ_TB_VSL_BSC |

## DBMS별(Oracle 11g) 추가 점검

| 항목 | 점검 | 판정 |
|---|---|---|
| 식별자 길이(30byte) | 최장 `MAX_EMKT_PRSNL_SUM`(18) 등 전부 30 이내 | 적합 |
| 타입 호환 | VARCHAR2/NUMBER/DATE 매핑 정합 | 적합 |
| 채번 | SEQ_TB_VSL_BSC 설계 | 적합 |
| 페이징 전제 | ROWNUM 가능 구조 | 적합 |

## 종합 판정

- **적합** — 적재 진행 가능. 단, 아래 2건은 **보완(설계 의도 반영)**으로 ADR/품질대장에 명시:
  1. 반복그룹 비정규화(원천 1:1 적재 목적) — 정규화는 후속 과제
  2. 범주형 컬럼 표준코드 미적용 — 원천값 보존, 코드화는 후속 과제
- 게이트: 구조 건전성 `적합`이므로 4단계(산출물)·적재 진행. 보완 2건은 품질부적합대장(NCR)에 관찰항목으로 등록.
