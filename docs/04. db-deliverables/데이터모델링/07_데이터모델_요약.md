# 07. 데이터모델 요약 — 사용자디렉토리/통합인증

- 대상 DBMS: Oracle 11g / 스키마: KOMSA
- 객체: 테이블 8종 + 뷰 1종
- 원천: eGov 공통컴포넌트 v2.0 사용자/권한 관리 객체(XLS 분석)

### 텍스트 ERD (계층)

```
[조직정보] 1 ──< N [업무사용자정보] 1 ──┬── 1 [사용자보안설정]
                                        ├── < N [업무사용자정보변경내역]
                                        └── 1 [로그인정책]

[메뉴정보] 1 ──< N [메뉴정보]  (UPPER_MENU_NO 자기참조)

[일반회원] ┐
[기업회원] ├──> (UNION ALL) ──> [사용자마스터뷰 COMVNUSERMASTER]
[업무사용자정보] ┘
```

## 2. 관계 목록

### 외부 주제영역 참조(본 DDL 제약 제외)
- `GROUP_ID` → 권한그룹정보(COMTNAUTHORGROUPINFO, 보안관리) — 외부, 보완 필요
- `AUTHOR_CODE` → 권한정보(COMTNAUTHORINFO, 보안관리) — 관리코드 아님, 보완 필요
- `PROGRM_FILE_NM` → 프로그램목록 — 원본 미정의, 보완 필요

## 3. 제약사항·검증 요약

### 키/무결성
- 단일 PK: COMTNEMPLYRINFO, COMTNMENUINFO, COMTNORGNZTINFO, COMTNENTRPRSMBER, COMTNGNRLMBER, COMTNEMPLYRSCRTYESTBS, COMTNLOGINPOLICY
- 복합 PK: COMTHEMPLYRINFOCHANGEDTLS(EMPLYR_ID + CHANGE_DE) — 이력 스냅샷
- 생성된 FK: 3건(R1, R2, R4). 논리/외부 참조 3건은 제약 미생성(보완 필요로 명시).

### 코드성 컬럼(공통코드 연계)
| 코드 | 컬럼 | 도메인 |
|---|---|---|
| CDK-COM-013 | EMPLYR_STTUS_CODE | 코드C1 |
| CDK-COM-014 | SEXDSTN_CODE | 코드C1 |
| CDK-COM-025 | PSTINST_CODE | 코드C8 |
| CDK-COM-026 | ENTRPRS_SE_CODE | 코드C8 |
| CDK-COM-027 | INDUTY_CODE | 코드C1 |
| CDK-COM-012 | MBER_TY_CODE | 코드C5 |

### 개인정보·보안
- 암호화 저장(SHA-256): PASSWORD, IHIDNUM, APPLCNT_IHIDNUM, ENTRPRS_MBER_PASSWORD → `VARCHAR2(200)`
- 개인정보 포함 테이블: 업무사용자정보, 기업회원, 일반회원, 업무사용자정보변경내역, (뷰)사용자마스터뷰
- 뷰 COMVNUSERMASTER는 비밀번호(해시)·이메일을 노출하므로 접근권한 통제 필요.

### Oracle 11g 호환 점검
- 모든 타입 VARCHAR2/CHAR/NUMBER/DATE — 11g 호환.
- 페이징은 ROWNUM 사용(애플리케이션 SQL). 본 DDL에는 페이징 미포함.
- 뷰는 UNION ALL 사용(중복 제거 불필요, 회원군 상호 배타).

### 보완 필요 항목(추측 금지 표기)
1. GROUP_ID/AUTHOR_CODE/PROGRM_FILE_NM 외부 참조 대상 테이블이 본 주제영역에 부재 → FK 제약 미생성, 연계 시 확정 필요.
2. COMTNLOGINPOLICY.EMPLYR_ID, COMTNEMPLYRSCRTYESTBS.SCRTY_DTRMN_TRGET_ID의 COMTNEMPLYRINFO 참조는 원본 FK 명시 없음 → 논리 참조로만 표기.
3. COMTNEMPLYRSCRTYESTBS 인덱스 I01/I03/I04(SCRTY_DTRMN_TRGET_ID 선두)는 PK와 중복 가능 → 본 DDL은 비중복 인덱스(I02: AUTHOR_CODE)만 생성. 운영 요구 시 재검토.
4. 코드성 컬럼의 행정표준코드 매핑은 표준화검토(metadata-agent)에서 확정.
5. 비밀번호/주민번호 길이 200 vs 표준도메인 `암호화번호V256` 차이 → 원본 유지(eGov 표준 준수) 결정.

## 4. 통합인증 관점 요약
- 3종 사용자(업무/일반/기업)를 분리 저장하되, `COMVNUSERMASTER` 뷰로 단일 사용자 디렉토리를 제공한다.
- 인증 기준 식별자: `USER_ID`(테이블별 PK 통합) + `USER_SE`(GNR/ENT/EMP) + `ESNTL_ID`(고유ID).
- 권한 결정: `COMTNEMPLYRSCRTYESTBS`(보안설정대상ID=사용자ID) → AUTHOR_CODE → 보안관리 주제영역의 권한·롤 모델과 연계.
- 접근 통제: `COMTNLOGINPOLICY`(IP/중복허용/제한), 디렉토리: `COMTNORGNZTINFO`(조직) + `COMTNMENUINFO`(메뉴).
