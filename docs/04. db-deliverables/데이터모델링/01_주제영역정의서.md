# 01. 주제영역정의서 — 사용자디렉토리/통합인증

## 개요
- 대상 DBMS: **Oracle 11g**
- 업무영역: 공통(COM) / 주제영역: **사용자디렉토리·통합인증(User Directory / Integrated Authentication)**
- 원천: 전자정부 표준프레임워크(eGov) 공통컴포넌트 v2.0 사용자/권한 관리 객체 정의(XLS 분석 결과)
- 스키마: **KOMSA**
- 영문 물리명 정책: 원본 테이블ID/컬럼ID는 **eGov 공통컴포넌트 표준 객체명**이므로 영문 물리명을 **그대로 유지**한다(표준 준수). 표준화 검토는 한글 용어·도메인 매핑·검토의견 위주로 수행한다.

## 주제영역 정의

| 항목 | 내용 |
|---|---|
| 주제영역ID | SA-USER-AUTH |
| 주제영역명 | 사용자디렉토리/통합인증 |
| 설명 | 업무사용자·일반회원·기업회원 등 시스템 이용 주체의 신원 정보, 조직/메뉴 디렉토리, 사용자 보안설정(권한), 정보변경 이력, 로그인 정책을 관리하는 주제영역. 세 종류 회원을 단일 뷰(사용자마스터뷰)로 통합 조회하여 통합인증의 기준 데이터를 제공한다. |
| 관리부서 | 정보화 담당(통합인증/계정관리) |
| 개인정보 포함 | 포함(주민등록번호·비밀번호·연락처·주소·이메일 등 — 일부 SHA-256 암호화 저장) |

## 관계 요약

| # | 부모 엔터티 | 자식 엔터티 | 관계 | 카디널리티 | FK 컬럼 |
|---|---|---|---|---|---|
| R1 | 조직정보 | 업무사용자정보 | 조직은 여러 업무사용자를 가진다 | 1:N | COMTNEMPLYRINFO.ORGNZT_ID → COMTNORGNZTINFO.ORGNZT_ID |
| R2 | 메뉴정보 | 메뉴정보(상위) | 메뉴는 상위 메뉴를 가진다(자기참조) | 1:N | COMTNMENUINFO.UPPER_MENU_NO → COMTNMENUINFO.MENU_NO |
| R3 | 업무사용자정보 | 사용자보안설정 | 업무사용자는 보안설정을 가진다 | 1:1 | COMTNEMPLYRSCRTYESTBS.SCRTY_DTRMN_TRGET_ID → COMTNEMPLYRINFO.EMPLYR_ID |
| R4 | 업무사용자정보 | 업무사용자정보변경내역 | 업무사용자는 여러 변경내역을 가진다 | 1:N | COMTHEMPLYRINFOCHANGEDTLS.EMPLYR_ID → COMTNEMPLYRINFO.EMPLYR_ID |
| R5 | 업무사용자정보 | 로그인정책 | 업무사용자는 로그인정책을 가진다 | 1:1 | COMTNLOGINPOLICY.EMPLYR_ID → COMTNEMPLYRINFO.EMPLYR_ID (보완 필요: 원본 FK 명시 없음) |
| R6 | (업무/일반/기업 회원) | 사용자마스터뷰 | 3종 회원을 UNION으로 통합 조회 | N:1(뷰) | COMVNUSERMASTER ← COMTNGNRLMBER / COMTNENTRPRSMBER / COMTNEMPLYRINFO |

### 참조 관계 보완 사항(추측 금지 표기)
- **GROUP_ID(그룹ID)**: COMTNEMPLYRINFO / COMTNENTRPRSMBER / COMTNGNRLMBER에 FK로 표기되어 있으나, 원본(XLS) 테이블 목록에 그룹 마스터 테이블이 포함되지 않음 → **외부 권한그룹정보(COMTNAUTHORGROUPINFO, 보안관리 주제영역) 참조로 추정**. 본 주제영역 내 대상 테이블 부재로 `보완 필요`.
- **AUTHOR_CODE(권한코드)**: COMTNEMPLYRSCRTYESTBS의 FK로 표기되나 "관리되는 코드가 아님"(자유식별값). 권한정보(COMTNAUTHORINFO, 보안관리)를 참조 → 본 주제영역 외부, `보완 필요`.
- **PROGRM_FILE_NM(프로그램파일명)**: COMTNMENUINFO의 FK로 표기되나 원본 목록에 프로그램목록 테이블 부재 → `보완 필요`.
- 위 외부 참조 대상은 원본에 정의되지 않았으므로 DDL의 FK 제약에는 포함하지 않고, **검토의견으로만 기록**한다.
